# Xmobile: Добавлен публичный портал и премиальный дизайн
Система была успешно расширена и доработана. Теперь это не только внутренняя ERP/CRM-система для сотрудников, но и полноценный, красивый клиентский портал. Все изменения реализованы без добавления сложных фронтенд-фреймворков (использованы только возможности html/template, Tailwind CSS и Vanilla JS), что сохраняет приложение быстрым и легким.

Что было добавлено и улучшено
1. Публичная часть (Клиентский портал)
Мы освободили корень сайта / от внутренней CRM (которая теперь доступна по адресу /orders) и разместили там премиальный лендинг для клиентов сервисного центра.

Лендинг-пейдж (/):
Современный дизайн с эффектом матового стекла (glassmorphism), фоновыми градиентными подсветками и микро-анимациями.
Описание преимуществ сервисного центра.
Интегрированная форма для быстрой отправки заявки на ремонт.
Страница отслеживания (/track):
Полноценная система отслеживания статуса заказа.
Для безопасности и защиты персональных данных клиентов, статус можно узнать только по связке "Номер заявки + Номер телефона".
Статусы подсвечиваются разными цветами (например, готовый заказ светится зеленым, новый — фиолетовым, в работе — синим).
Экран успеха (/request/success): После отправки формы клиент получает красивое подтверждение с автоматически сгенерированным номером его заявки.
2. Изменения в CRM и Базе Данных
Добавлен новый статус заказа: StatusNew («Новая заявка»). Теперь все заказы с сайта по умолчанию попадают в базу именно под этим статусом. Мастера в CRM могут сразу отличать онлайн-заявки от устройств, которые принесли вживую.
Внутренняя система переехала на /orders. Навигация и редиректы после входа в систему (/login) были автоматически обновлены для сотрудников.
Инструкция по запуску
Проект по-прежнему легко запускается как локально, так и через Docker:

Через Docker Compose (Рекомендуется)
bash

cd /home/student/mobile-service
docker compose up --build -d
Сайт будет доступен по адресу http://localhost:8080.

Локальный запуск через Go
bash

cd /home/student/mobile-service
go mod tidy
go build ./...
mkdir -p data
go run ./cmd/app
